# 项目目标

## 一、项目愿景

CampusShareOrder 旨在构建一个面向校园场景的拼单平台，解决学生日常生活中的拼单需求，提供统一、高效、可信的拼单管理解决方案。

## 二、核心价值

### 1. 解决痛点
- **信息整合**：将分散在各群组的拼单信息集中管理，方便查找和参与
- **状态透明**：实时展示拼单状态，避免错过重要信息
- **资金清晰**：通过伪支付系统记录交易，减少转账混乱和纠纷
- **信用机制**：建立信用分体系，降低"放鸽子"风险
- **纠纷处理**：提供投诉渠道和处理机制，保障用户权益

### 2. 提升体验
- **统一平台**：一站式管理所有拼单活动
- **流程闭环**：从创建到收货的完整业务流程
- **及时通知**：关键节点状态变更提醒
- **便捷操作**：移动端适配，随时随地参与拼单

## 三、功能目标

### 用户端核心功能
1. **用户管理**
   - 用户注册与登录
   - 学号实名认证
   - 个人信息管理
   - 信用分展示与查询

2. **拼单管理**
   - 发起拼单（设置人数上限、截止时间）
   - 浏览拼单列表（支持筛选和搜索）
   - 查看拼单详情
   - 加入拼单 / 退出拼单（未成团状态）
   - 查看我的拼单

3. **订单流程**
   - 伪支付（模拟支付流程，不涉及真实资金）
   - 自动成团（达到人数上限或截止时间）
   - 上传订单凭证
   - 确认送达（发起人确认）
   - 确认收货（参与者确认，支持自动确认）
   - 拼单完成

4. **信任与投诉**
   - 信用分计算与展示
   - 提交投诉（针对发起人或参与者）
   - 投诉进度查询
   - 投诉结果通知

### 管理端核心功能
1. **用户管理**
   - 查看用户列表
   - 封禁 / 解封用户
   - 查看用户信用分历史

2. **拼单管理**
   - 查看所有拼单
   - 查看拼单详情
   - 处理异常拼单

3. **投诉处理**
   - 查看投诉列表
   - 处理投诉（同意/驳回）
   - 退款处理
   - 扣分处理

4. **数据查看**
   - 支付 / 退款记录
   - 操作日志查看
   - 系统统计（拼单数量、用户活跃度等）

## 四、业务流程目标

### 正常流程
```
创建拼单 → 加入拼单 → 支付 → 成团 → 上传凭证 → 确认送达 → 确认收货 → 完成
```

### 异常流程
```
未成团 → 自动取消 → 自动退款
发起人异常 → 投诉 → 管理员处理 → 退款 + 扣分
```

## 五、技术目标

### 1. 架构目标
- 前后端分离架构
- RESTful API 设计
- 模块化、可扩展的代码结构

### 2. 性能目标
- 支持并发用户访问
- 页面响应时间 < 2秒
- 接口响应时间 < 500ms

### 3. 安全目标
- JWT 认证机制
- 权限控制（用户/管理员）
- 数据验证与防注入
- 操作日志记录

### 4. 可维护性目标
- 清晰的代码注释
- 统一的代码风格
- 完善的错误处理
- 便于部署和扩展

## 六、开发周期目标

### 第一次迭代（第1-7天）
- 用户注册与登录
- 实名认证
- 拼单创建 / 加入 / 退出
- 拼单列表 / 详情展示
- 伪支付 / 伪退款
- 自动成团 / 自动取消

### 第二次迭代（第8-14天）
- 上传订单凭证
- 确认送达
- 确认收货
- 投诉处理
- 信用分更新
- 管理后台

## 七、成功标准

1. **功能完整性**：所有核心功能正常运行，业务流程闭环
2. **用户体验**：界面友好，操作流畅，移动端适配良好
3. **代码质量**：代码规范，注释清晰，易于维护
4. **文档完善**：需求文档、设计文档、接口文档齐全
5. **演示效果**：能够成功演示完整的拼单流程
6. **课程要求**：满足软件工程课程设计验收标准

## 八、简化原则

- **伪支付/伪退款**：不涉及真实资金，重点在流程逻辑
- **简化存储**：使用本地文件存储凭证图片
- **简化部署**：优先本地开发环境部署，简化配置
- **优先流程**：优先保证业务流程闭环，而非性能优化
- **适应周期**：两周开发周期，合理控制功能范围

## 九、未来扩展方向

1. **功能扩展**
   - 真实支付集成（微信支付/支付宝）
   - 实时消息通知（WebSocket）
   - 拼单推荐算法
   - 积分系统与奖励机制

2. **技术优化**
   - Redis 缓存优化
   - 数据库分库分表
   - 微服务架构改造
   - 自动化测试覆盖

3. **运营支持**
   - 数据分析仪表板
   - 用户行为分析
   - A/B 测试框架
   - 运营工具集成

## 十、项目定位

CampusShareOrder 是一个软件工程课程设计项目，定位为：
- **教学项目**：展示软件工程实践能力
- **原型系统**：验证拼单平台可行性
- **学习平台**：学习和实践现代 Web 开发技术栈
- **演示系统**：能够进行完整的业务流程演示

通过本项目的开发，团队成员将在两周内完成一个具有完整业务闭环的 Web 应用，掌握前后端分离开发、RESTful API 设计、数据库设计、前端组件化开发等核心技能。
