# 校园拼单平台数据库设计方案

## 1. 数据库设计原则

### 1.1 设计理念

- **规范化设计**：遵循第三范式，减少数据冗余
- **性能优先**：合理索引设计，优化查询性能
- **扩展性**：预留字段和表结构，支持未来业务扩展
- **安全性**：敏感数据加密存储，权限控制

### 1.2 技术选型

- **数据库**：MySQL 8.0+
- **字符集**：utf8mb4 (支持emoji)
- **排序规则**：utf8mb4\_unicode\_ci
- **存储引擎**：InnoDB (支持事务和外键)

## 4. 设计优化与补充

### 4.1 索引优化

- **组合索引**：为常用查询场景创建组合索引，如订单状态+时间、用户ID+时间等
- **唯一索引**：确保业务唯一性，如手机号、学号、订单号等
- **外键索引**：所有外键字段都建立索引，提高关联查询性能

### 4.2 安全性考虑

- **密码存储**：使用BCrypt算法加密存储密码
- **敏感信息**：手机号、学号等敏感信息在API返回时进行脱敏处理
- **权限控制**：通过JWT和数据库状态双重验证确保权限安全

### 4.3 性能优化

- **分表策略**：对于操作日志等增长较快的表，预留分表方案
- **缓存策略**：对于热点数据（如用户信息、订单状态）考虑使用缓存
- **查询优化**：合理使用索引，避免全表扫描

### 4.4 扩展性考虑

- **预留字段**：在核心表中预留适量扩展字段
- **状态枚举**：使用字符串枚举类型，便于后续扩展新状态
- **JSON字段**：使用JSON类型存储非结构化数据，提高灵活性

### 5.2 业务规则

| 规则     | 说明                                                               |
| :----- | :--------------------------------------------------------------- |
| 成团条件   | currentMemberCount == totalMemberCount 且所有ACTIVE成员payStatus=PAID |
