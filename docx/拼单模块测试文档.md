# 拼单模块接口测试方法

## 1. 创建拼单接口

**URL**: `http://localhost:8080/api/orders`\
**方法**: `POST`\
**请求头**: `Content-Type: application/json`

**请求参数**:

| 参数名                  | 类型         | 必填 | 说明          |
| :------------------- | :--------- | :- | :---------- |
| productName          | String     | 是  | 商品名称        |
| productDesc          | String     | 否  | 商品描述        |
| totalMemberCount     | Integer    | 是  | 总人数（至少2人）   |
| estimatedTotalAmount | BigDecimal | 是  | 预估总金额（大于0）  |
| pickupPoint          | String     | 是  | 取货点         |
| deadlineAt           | String     | 是  | 截止时间（ISO格式） |

**示例请求**:

```json
{
  "productName": "奶茶拼单",
  "productDesc": "三分糖，少冰",
  "totalMemberCount": 3,
  "estimatedTotalAmount": 30.0,
  "pickupPoint": "图书馆门口",
  "deadlineAt": "2026-04-20T12:00:00"
}
```

**预期响应**:

```json
{
  "code": 200,
  "message": "成功",
  "data": {
    "orderId": 1,
    "orderNo": "ORDER1234567890AB",
    "status": "拼单中"
  }
}
```

## 4. 加入拼单接口

**URL**: `http://localhost:8080/api/orders/{orderId}/join`\
**方法**: `POST`\
**路径参数**: `orderId` - 订单ID\
**请求头**: `Content-Type: application/json`

**请求参数**:

| 参数名    | 类型     | 必填 | 说明   |
| :----- | :----- | :- | :--- |
| remark | String | 否  | 备注信息 |

**示例请求**:

```json
{
  "remark": "五分糖，去冰"
}
```

**预期响应**:

```json
{
  "code": 200,
  "message": "成功",
  "data": null
}
```

## 测试注意事项

1. **用户ID**：当前所有操作都使用硬编码的用户ID 1，确保数据库中存在该用户
2. **状态验证**：测试过程中注意验证订单状态的变化
3. **错误处理**：测试错误场景，如加入已满员的订单、支付已支付的订单等
4. **数据库验证**：测试完成后检查数据库中的数据是否正确更新

## 测试工具推荐

- **Postman**：图形化API测试工具，支持请求管理和环境变量
- **curl**：命令行工具，适合自动化测试
- **Swagger**：如果项目集成了Swagger，可以直接在浏览器中测试

通过以上测试方法，您可以全面验证拼单模块的核心功能是否正常工作。
