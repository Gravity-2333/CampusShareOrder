### 2.5 API 设计

#### 2.5.1 认证相关 API
- `POST /api/auth/register` - 用户注册
- `POST /api/auth/login` - 用户登录
- `POST /api/auth/verify-student` - 学号认证
- `POST /api/auth/refresh` - 刷新 Token

#### 2.5.2 用户相关 API
- `GET /api/user/profile` - 获取个人信息
- `PUT /api/user/profile` - 更新个人信息
- `GET /api/user/credit` - 获取信用分

#### 2.5.3 拼单相关 API
- `GET /api/orders` - 获取拼单列表
- `GET /api/orders/:id` - 获取拼单详情
- `POST /api/orders` - 创建拼单
- `POST /api/orders/:id/join` - 加入拼单
- `POST /api/orders/:id/leave` - 退出拼单
- `POST /api/orders/:id/pay` - 支付
- `POST /api/orders/:id/upload-proof` - 上传凭证
- `POST /api/orders/:id/confirm-delivery` - 确认送达
- `POST /api/orders/:id/confirm-receipt` - 确认收货

#### 2.5.4 投诉相关 API
- `GET /api/complaints/my` - 获取我的投诉
- `POST /api/complaints` - 创建投诉
- `GET /api/complaints/:id` - 获取投诉详情

#### 2.5.5 管理员相关 API
- `POST /api/admin/login` - 管理员登录
- `GET /api/admin/users` - 获取用户列表
- `PUT /api/admin/users/:id/ban` - 封禁用户
- `PUT /api/admin/users/:id/unban` - 解封用户
- `GET /api/admin/orders` - 获取所有拼单
- `GET /api/admin/complaints` - 获取投诉列表
- `PUT /api/admin/complaints/:id/handle` - 处理投诉
- `GET /api/admin/payments` - 获取支付记录
- `GET /api/admin/refunds` - 获取退款记录
- `GET /api/admin/logs` - 获取操作日志

## 六、架构优势

### 6.1 前端架构优势
- **组件化**：Vue 3 组件化开发，代码复用性高
- **响应式**：Pinia 状态管理，数据流清晰
- **类型安全**：JavaScript 静态检查，减少错误
- **开发效率**：Vite 快速构建，Element Plus 丰富的组件

### 6.2 后端架构优势
- **成熟框架**：Spring Boot 生态成熟，文档丰富
- **安全可靠**：Spring Security 提供完善的安全机制
- **易于扩展**：分层架构，便于添加新功能
- **ORM 简化**：MyBatis Plus 简化数据库操作

### 6.3 整体架构优势
- **前后端分离**：职责清晰，便于并行开发
- **RESTful API**：接口标准化，易于对接
- **技术栈现代**：使用主流技术，学习资源丰富
- **适合课程**：两周开发周期内可完成核心功能
