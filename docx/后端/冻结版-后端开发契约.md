# 校园拼单平台冻结版后端开发契约

本文档作为当前仓库后端开发的正式基线，用于统一包结构、接口口径、DTO/VO、实体设计、状态机、错误码、定时任务和验收用例。

适用范围：

- 后端：`backend`
- 当前阶段：课程项目第一版
- 对齐基线：前端冻结契约

统一原则：

- 接口统一前缀为 `/api`
- 成功响应统一为 `{ "code": 0, "message": "success", "data": ... }`
- 分页响应统一为 `{ "list": [], "page": 1, "pageSize": 10, "total": 0, "pages": 0 }`
- DTO / VO 的 JSON 字段名必须与前端完全一致
- 所有状态名、接口名、字段名只保留一套
- 当前版本只允许使用 `@Scheduled + Service + @Transactional`
- V1 不引入 MQ，不做真实支付，不做通知中心页面

### 6.1 `actionFlags` 最终字段

固定为：

- `canJoin`
- `canPay`
- `canExit`
- `canUploadReceipt`
- `canMarkDelivered`
- `canConfirmReceived`
- `canCreateComplaint`

规则：

- 前端按钮展示唯一信源是 `actionFlags`
- 前端不得再自行推导复杂业务按钮

## 9. 安全与鉴权冻结方案

### 9.1 放行接口

- `POST /api/auth/register`
- `POST /api/auth/login`
- `POST /api/auth/admin/login`
- `GET /uploads/**`

### 9.2 `USER` 必须登录

- `GET /api/auth/me`
- 所有 `/api/users/**`
- 所有普通订单接口 `/api/orders/**`
- 所有普通投诉接口 `/api/complaints/**`

### 9.3 `ADMIN` 必须登录

- 所有 `/api/admin/**`

### 9.4 固定规则

- JWT 必须带 `role`
- `GET /api/auth/me` 按 `role` 决定查 `user_account` 或 `admin_account`
- `USER` token 不允许访问 `/api/admin/**`
- `ADMIN` token 不允许访问普通用户业务接口
- 被封禁用户统一返回 `40302`

## 10. 状态机冻结规则

### 10.1 主状态

- `OPEN`
- `GROUPED`
- `WAIT_DELIVERY`
- `WAIT_RECEIVE`
- `COMPLETED`
- `CANCELED`

### 10.2 成团规则

只有同时满足以下条件才成团：

- `currentMemberCount == totalMemberCount`
- 全部 `ACTIVE` 成员 `payStatus = PAID`
- 发起人也必须支付

### 10.3 关键拍板

- 差额退款发生在上传凭证时，不改变主状态
- 成团后 `30 分钟` 未上传凭证只开放投诉，不直接取消
- 超过预计最晚送达时间 `60 分钟` 只开放投诉，不直接取消
- 自动确认收货时间统一为 `30 分钟`
- 投诉成立后订单固定进入 `CANCELED`
- 自动确认收货后立即尝试完成订单

## 11. 事务边界冻结

以下动作必须放在事务内：

- 创建订单
- 加入订单
- 支付
- 最后一人支付触发成团
- 退出订单
- 上传凭证
- 差额退款
- 确认送达
- 确认收货
- 自动确认收货
- 自动完成订单
- 投诉提交
- 投诉处理
- 管理员取消订单

关键规则：

- 退款和结算必须做幂等
- 取消退款统一按 `payAmount - refundAmountTotal` 计算净退款
- 完成订单结算只能生成一次 `SETTLE_TO_CREATOR`
- 信用分加减必须避免重复写入

## 16. 前后端对接高风险点

必须重点避免：

1. JSON 字段名不一致
2. 时间格式不一致
3. 金额使用 `double` 导致精度错误
4. 前后端状态判断不一致
5. 分页结构不一致
6. `actionFlags` 推导不一致
7. 错误码不一致
8. 用户与管理员 token 混用
9. 退款重复生成
10. 自动任务和手动操作并发冲突

固定预防规则：

- DTO / VO 字段名只按冻结契约
- 时间字段统一输出 `yyyy-MM-dd HH:mm:ss`
- 金额统一使用 `BigDecimal`
- 前端按钮唯一信源是 `actionFlags`
- 退款、结算、加分、扣分必须幂等
- 定时任务执行前必须二次校验数据库最新状态
